# Fleet v9 사이클 결과 보고 — 마우스로 세션 고르고 죽이기 · 서브에이전트 관측

날짜: 2026-07-15 · 대상: `tools/fleet` (+ `adapters/claude/tools/fleet` 미러)

## 결론

병합 가능. 이번 사이클에서 만든 두 가지 기능 — **마우스로 행을 클릭해 세션을 고르고 다시 클릭해 죽이는 경로**, **하위로 뻗어나간 서브에이전트를 세션 트리 안에서 보여주는 관측 기능** — 이 계획대로 동작하고, 회귀도 없다. 독립 검증자가 전체 항목(22개 acceptance 기준)을 직접 재실행해 확인했고, 블로킹 결함은 0건이다. 다만 진행 중 발견된 사양 이탈 1건이 있었고, 검증 단계에서 되돌려졌다 — 아래에 그대로 기록한다.

## 무엇이 바뀌었나

### 1. 마우스로 세션 선택 · 종료 (`prd.md:278-284`)
- 세션 행을 클릭 → 선택. 같은 행을 다시 클릭 → 종료 확인 요청. `[kill]`/`[cancel]` 클릭 타깃 각각 종료 확정/취소. 행 밖 클릭 → 선택 해제.
- 기존 키보드 조작(`s`로 선택, `x`로 종료)은 완전히 그대로 유지된다.
- 설계상 까다로웠던 지점 두 가지를 코드로 못 박았다(계획 리뷰가 "어기면 조용히 안전이 깨진다"고 표시한 부분):
  - 더블클릭으로 인한 오검(재그리기 없이 두 번째 클릭이 들어와 확인 단계가 무력화되는 문제)을 막기 위해, 마우스 처리는 반드시 화면 재그리기 앞 블록 안에서만 일어나도록 강제.
  - 클릭 가능 영역 판정은 매 프레임이 아니라 클릭이 들어온 순간에만 계산 — 초당 10회 반복 판정으로 인한 불필요한 부하를 피했다.
- 독립 검증자가 자체 프로브로 6가지 폭 × 3단계 조합을 전수 확인 — 같은 좌표를 연타해도 종료로 직행할 수 없음을 실증했다.

### 2. 서브에이전트 관측 (`prd.md:290-295`)
- 세션 목록에 원래 있던 세션 밑으로, 그 세션이 분사한 서브에이전트를 연결된 트리 형태(`├⚡`/`└⚡`)로 붙여 보여준다. 배지(`⚡N`)로 개수도 표시.
- 소스는 OpenCode의 자식 세션 정보와 Claude의 사이드체인 정보 두 가지. Codex는 아직 신호가 없어 표시하지 않는다 — 없는 걸 있는 척 만들지 않고 정직하게 결손으로 남겼다.
- 기존 세션 목록·`--json` 출력에는 순수 추가만 했다(`subagents` 키 신설, 기존 키 무변경). 서브에이전트 조회가 실패해도 원래 세션 행 표시에는 영향 없음을 예외 주입으로 확인.

### 3. 잔여 후속 처리
- dispatch 트리의 폭이 168칸에서도 구조적으로 넘치지 않도록 폭 상한을 단일 상수로 고정.
- `test_arrow_keys_still_scroll_in_base_mode` 신설로 "마우스 도입이 화살표 스크롤을 깨지 않는다"를 회귀 테스트로 못 박음.

## 어떻게 검증했나

- 전체 회귀: `python3 -m unittest discover -s tools/fleet/tests -q` → 베이스라인 416 → **468 통과**(신규 52 = 마우스 21 + 서브에이전트 15 + 폭 상한 6 + 스크롤 10).
- 미러 동기화: `rsync -a --delete tools/fleet/ adapters/claude/tools/fleet/` 후 `diff -r tools/fleet adapters/claude/tools/fleet` → 무출력(바이트 단위 동일).
- 3폭 렌더 육안 확인: `COLUMNS={60,120,168} python3 tools/fleet/fleet.py --once`.
- `--json` 스모크: `subagents` 키 추가 확인, 기존 키 무변화.
- 독립 검증자(별도 워커, execute의 주장을 신뢰하지 않고 전부 재실행)가 22개 acceptance 항목 전체를 PRD 절 대조로 판정 — **전부 PASS**.
- 안전 계약: 사이클 전체에서 실제 `claude`/`codex`/`opencode` 세션을 스폰하거나 시그널을 보낸 적 0회. 전부 fixture·mock(`curses.getmouse`, `os.kill` monkeypatch, `sleep` 프로세스)로만 검증. 마우스 이벤트는 curses `getmouse`를 통해 완전히 격리된 상태로 모의됐다.

## 글리프 이탈 — 있었던 일 그대로

구현 단계에서 서브에이전트 배지 아이콘을 PRD가 정한 `⚡` 대신 `🔬`로 바꿔 넣었다. 근거로 든 건 "`⚡`가 이미 `⚡untracked` 게이트 배지에 쓰이고 있어 충돌한다"는 주장이었다.

독립 검증 단계에서 이 주장을 직접 코드로 확인한 결과 **거짓으로 판정**됐다: fleet은 어떤 폭에서도 `⚡`를 실제로 렌더링한 적이 없고(`_gate_word`는 문자열 "untracked"만 반환, 글리프는 statusline/hooks 쪽 별개 표면에만 존재), 애초에 계획 리뷰(plan.md:347)가 "`⚡`는 신설이고 충돌 없음"이라고 이미 판정했던 것을 구현 단계가 반박 없이 뒤집은 것이었다. 검증자는 원복을 권고했고, 지휘 세션이 이를 재현 확인한 뒤 `render.py`의 2줄(상수 + 주석)을 되돌리고 미러를 재동기화했다. 되돌린 뒤 468 전체 재통과, 미러 바이트 일치 재확인.

결과적으로 코드는 PRD(`prd.md:293`)와 다시 일치하며, PRD 개정이나 마이너 로그 기록은 필요 없다. 이 에피소드 자체가 파이프라인이 의도대로 작동한 사례라고 보고한다 — 구현 단계의 근거 없는 이탈을 검증 단계가 잡아냈다.

## 아직 열려 있는 것 (다음 사이클 후보)

- **OpenCode 서브에이전트가 항상 "진행 중"으로 보임**: OpenCode 쪽 데이터에 완료 신호가 없어 "완료된 것은 숨긴다"는 목표를 이 소스에서는 달성할 수 없다. 정직하게 문서화는 돼 있으나, 배지 카운트가 늘어나기만 할 수 있다는 한계가 남는다.
- **미러 트리를 직접 실행하면 3개 테스트 실패**: 이번 사이클 변경과 무관 — 현재 main(468 이전, 416 기준)에서도 동일하게 재현되는 사전 존재 결함(테스트가 저장소 루트를 거슬러 올라가다 미러 위치에서 오탐지). 별도 이슈로 분리 권고.
- **60칸 폭에서 legend/mem/usage 요약 행 미축약**: 이번 사이클 범위 밖(D4)으로 원래도 알려진 결함. 원인이 이번에 고친 dispatch 폭 상한 문제와 다르다.
- 결정 필요한 항목 없음 — 위 세 가지는 모두 "알고 있고, 다음으로 미룬다"는 상태.
